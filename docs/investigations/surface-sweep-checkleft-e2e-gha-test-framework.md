## Recommendation

- **No restoration task to file.** There is no regressed or deleted _shipped_ surface to restore.
- **Re-run this sweep after implementation lands.** The meaningful surface sweep for this project happens once design PR #1470 merges and tasks **T1, T4, T5, T7–T9** (the ones that ship user-facing surfaces) have been implemented and merged to `main`. At that point S1–S8 become real, checkable surfaces.
- **Optional design hygiene (out of scope for this investigation):** the design doc has no `§Surfacing` section, which is what makes an automated surface sweep unable to key off a canonical list. If this project is meant to participate in the incident-002 backstop, adding a short `§Surfacing` enumeration (the S1–S8 rows above are a ready-made starting point) would let future sweeps run mechanically rather than reconstruct the list from the task breakdown. This is a docs change on the design PR, filed as a follow-up rather than done here.
